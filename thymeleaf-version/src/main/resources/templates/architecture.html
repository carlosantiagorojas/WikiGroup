Arquitectura del Proyecto

Modelos y diagramas para el desarrollo del proyecto

Modelo conceptual

El modelo conceptual describe la estructura y las relaciones clave de un sistema de gestión de música en línea. Incluye entidades como "Usuario", "Canción", "Lista de Reproducción" y "Administrador". Los usuarios pueden "dar like" a las canciones, y los administradores pueden crear y gestionar listas de reproducción. Las relaciones principales son "Usuario da Like a Canción" y "Administrador crea Lista de Reproducción". Este modelo ayuda a comprender la lógica básica del sistema.

Modelo contenedores

En el contexto de contenedores, se pueden usar tecnologías como angular para frontend y SpringBoot para backend para implementar el sistema de gestión de música en línea. Cada componente del sistema, como la aplicación web, la base de datos y los servidores de aplicación, se empaqueta en contenedores independientes. Esto facilita la escalabilidad, la administración y la implementación eficiente.

Modelo componente

El componente de una página web en el sistema de gestión de música en línea incluye todos los elementos que interactúan para brindar una experiencia completa a los usuarios. En el frontend, se encuentran los elementos de la interfaz de usuario, como botones de "like", listas de reproducción, reproductor de música y formularios de registro/inicio de sesión. Estos elementos se implementan utilizando tecnologías web como HTML, CSS y JavaScript. En el backend, hay componentes que gestionan la lógica del sistema, como la autenticación de usuarios, la gestión de listas de reproducción, la gestión de canciones y la interacción con la base de datos. Estos componentes están escritos en lenguajes de programación como Python, Ruby o Node.js y se ejecutan en servidores web. El despliegue implica la implementación de estos componentes en servidores, ya sea en una infraestructura local o en la nube utilizando servicios como AWS, Azure o Google Cloud.

Modelo de la base de datos

Estamos utilizando MySQL como nuestra base de datos. Utilizamos MySQL Workbench para visualizar y diseñar la base de datos, y también con Spring Boot generamos el esquema de la base de datos a partir de nuestros modelos. Implementamos algunos disparadores y procedimientos almacenados para gestionar lógica directamente en la base de datos. Finalmente, desplegamos la base de datos en AWS RDS.
