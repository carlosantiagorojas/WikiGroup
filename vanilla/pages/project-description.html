NUESTRO PROYECTO

El proyecto "MusicaList" se centra en el desarrollo de una aplicación web que busca satisfacer las necesidades de dos tipos de usuarios distintos: el usuario administrador del sistema y el usuario votante. El usuario administrador tiene la responsabilidad principal de crear, editar y eliminar géneros musicales y las canciones asociadas a ellos. Por otro lado, el usuario votante tiene la capacidad de crear y activar su cuenta, buscar canciones, verificar listas por género musical, votar y eliminar su voto en las canciones. Para obtener más detalles sobre las funcionalidades específicas de cada usuario, se pueden consultar las historias de usuario en la página web del proyecto.

El desarrollo de la plataforma implica la creación de un backend que exponga servicios REST utilizando la tecnología SPRING, y para la conexión con la base de datos se utilizará JPA. Además, se espera que los grupos de trabajo gestionen un repositorio de control de versiones para mantener un seguimiento adecuado del código y facilitar la colaboración entre los miembros del equipo. En la primera entrega, programada para el 26 de septiembre de 2023, se espera que se logre la conexión a la base de datos, la creación de modelos entidad-relación, el mapeo de modelos al proyecto JPA en Spring, la realización de pruebas unitarias con documentación detallada y la implementación de servicios CRUD para cada clase. También se espera que se presente una plantilla principal y de menú utilizando Thymeleaf, y que se desarrollen los métodos CRUD para cada clase con sus respectivos contratos funcionales.
